## <a name="how-does-microsoft-365-centralize-and-report-on-audit-logs"></a><span data-ttu-id="a1d6e-120">Microsoft 365 は監査ログの集中管理とレポート作成を行いますか?</span><span class="sxs-lookup"><span data-stu-id="a1d6e-120">How does Microsoft 365 centralize and report on audit logs?</span></span>

<span data-ttu-id="a1d6e-121">Microsoft 365 サーバーから、Cosmos と呼ばれる内部の大規模なデータコンピューティングサービスに、さまざまな種類のログデータがアップロードされています。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-121">Many different types of log data are uploaded from Microsoft 365 servers to an internal, big data computing service called Cosmos.</span></span> <span data-ttu-id="a1d6e-122">各サービスチームは、それぞれのサーバーからの監査ログを Cosmos データベースにアップロードして、集約と分析を行います。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-122">Each service team uploads audit logs from their respective servers into the Cosmos database for aggregation and analysis.</span></span> <span data-ttu-id="a1d6e-123">このデータ転送は、Office データローダー (ODL) と呼ばれる独自の自動化ツールを使用して、承認されたポートおよびプロトコルで、FIPS 140-2 で検証された TLS 接続を介して行われます。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-123">This data transfer occurs over a FIPS 140-2-validated TLS connection on approved ports and protocols using a proprietary automation tool called the Office Data Loader (ODL).</span></span>

<span data-ttu-id="a1d6e-124">サービスチームは、ログの関連付け、警告、およびレポート用に、Cosmos のデータに対してスコープ指定されたクエリを実行します。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-124">Service teams run scoped queries against their data in Cosmos for log correlation, alerting, and reporting.</span></span> <span data-ttu-id="a1d6e-125">たとえば、Microsoft 365 セキュリティチームは、独自のイベントログパーサーを使用して Cosmos のデータを使用して、ログデータの関連付け、通知の送信、および Microsoft 365 運用環境での疑わしい可能性のあるアクティビティに対する実践的なレポートの生成を行います。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-125">For example, the Microsoft 365 Security team uses data from Cosmos with a proprietary event log parser to correlate log data, send alerts, and generate actionable reports on possible suspicious activity in the Microsoft 365 production environment.</span></span> <span data-ttu-id="a1d6e-126">このデータからのレポートは、脆弱性を修正し、サービスの全体的なパフォーマンスを向上させるために使用されます。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-126">The reports from this data are used to correct vulnerabilities and to improve the overall performance of the service.</span></span>

## <a name="how-does-microsoft-365-protect-audit-logs"></a><span data-ttu-id="a1d6e-127">Microsoft 365 で監査ログを保護する方法</span><span class="sxs-lookup"><span data-stu-id="a1d6e-127">How does Microsoft 365 protect audit logs?</span></span>

<span data-ttu-id="a1d6e-128">Microsoft 365 で監査レコードを収集して処理するために使用されるツールでは、元の監査レコードのコンテンツや時間の順序を永続的に変更したり、取り消したりすることはできません。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-128">The tools used in Microsoft 365 to collect and process audit records do not allow permanent or irreversible changes to the original audit record content or time ordering.</span></span> <span data-ttu-id="a1d6e-129">Cosmos に格納されている Microsoft 365 データへのアクセスは、権限のある人物に制限されます。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-129">Access to Microsoft 365 data stored in Cosmos is restricted to authorized personnel.</span></span> <span data-ttu-id="a1d6e-130">Microsoft 365 では、監査機能の管理が、監査機能を担当するサービスチームメンバーの限定されたサブセットに制限されています。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-130">Microsoft 365 restricts the management of audit functionality to the limited subset of service team members that are responsible for audit functionality.</span></span> <span data-ttu-id="a1d6e-131">これらのチームメンバーは、Cosmos のデータを変更または削除することはできません。また、Cosmos のログメカニズムに加えられたすべての変更は記録および監査されます。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-131">These team members do not have the ability to modify or delete data from Cosmos, and all changes to logging mechanisms for Cosmos are recorded and audited.</span></span> <span data-ttu-id="a1d6e-132">監査ログは、インシデント調査をサポートし規制要件を満たすのに十分な期間保持されます。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-132">Audit logs are retained long enough to support incident investigations and meet regulatory requirements.</span></span> <span data-ttu-id="a1d6e-133">Cosmos での監査ログデータの保持期間の正確な期間は、サービスチームによって決まります。ほとんどの監査ログデータは、90日間またはそれ以上保持されます。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-133">The exact period of audit log data retention in Cosmos is determined by the service teams; most audit log data is retained for 90 days or longer.</span></span>

## <a name="how-does-microsoft-365-protect-end-user-identifiable-information-that-may-be-captured-in-audit-logs"></a><span data-ttu-id="a1d6e-134">Microsoft 365 は、監査ログに記録される可能性があるエンドユーザーの識別可能な情報を保護する方法を教えてください。</span><span class="sxs-lookup"><span data-stu-id="a1d6e-134">How does Microsoft 365 protect end-user identifiable information that may be captured in audit logs?</span></span>
