## <a name="how-does-microsoft-365-centralize-and-report-on-audit-logs"></a><span data-ttu-id="fa4c1-120">Microsoft 365 は監査ログを一元化して報告する方法を説明します。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-120">How does Microsoft 365 centralize and report on audit logs?</span></span>

<span data-ttu-id="fa4c1-121">Microsoft 365 サーバーから Cosmos と呼ばれる内部のビッグ データ コンピューティング サービスに、さまざまな種類のログ データがアップロードされます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-121">Many different types of log data are uploaded from Microsoft 365 servers to an internal, big data computing service called Cosmos.</span></span> <span data-ttu-id="fa4c1-122">各サービス チームは、それぞれのサーバーから Cosmos データベースに監査ログをアップロードして、集計と分析を行います。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-122">Each service team uploads audit logs from their respective servers into the Cosmos database for aggregation and analysis.</span></span> <span data-ttu-id="fa4c1-123">このデータ転送は、Office データ ローダー (ODL) と呼ばれる独自の自動化ツールを使用して、承認されたポートとプロトコル上の FIPS 140-2 検証済み TLS 接続を使用して行われます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-123">This data transfer occurs over a FIPS 140-2-validated TLS connection on approved ports and protocols using a proprietary automation tool called the Office Data Loader (ODL).</span></span>

<span data-ttu-id="fa4c1-124">サービス チームは、ログの相関関係、警告、レポート作成のために、Cosmos のデータに対して範囲指定されたクエリを実行します。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-124">Service teams run scoped queries against their data in Cosmos for log correlation, alerting, and reporting.</span></span> <span data-ttu-id="fa4c1-125">たとえば、Microsoft 365 セキュリティ チームは、Cosmos のデータと独自のイベント ログ パーサーを使用して、ログ データを関連付け、アラートを送信し、Microsoft 365 の実稼働環境での不審なアクティビティの可能性に関する操作可能なレポートを生成します。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-125">For example, the Microsoft 365 Security team uses data from Cosmos with a proprietary event log parser to correlate log data, send alerts, and generate actionable reports on possible suspicious activity in the Microsoft 365 production environment.</span></span> <span data-ttu-id="fa4c1-126">このデータからのレポートは、脆弱性を修正し、サービスの全体的なパフォーマンスを向上させるために使用されます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-126">The reports from this data are used to correct vulnerabilities and to improve the overall performance of the service.</span></span>

## <a name="how-does-microsoft-365-protect-audit-logs"></a><span data-ttu-id="fa4c1-127">Microsoft 365 は監査ログを保護する方法を説明します。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-127">How does Microsoft 365 protect audit logs?</span></span>

<span data-ttu-id="fa4c1-128">Microsoft 365 で監査レコードの収集と処理に使用されるツールでは、元の監査レコードのコンテンツや時間順序に対する永続的または取り返しのできない変更は許可されません。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-128">The tools used in Microsoft 365 to collect and process audit records do not allow permanent or irreversible changes to the original audit record content or time ordering.</span></span> <span data-ttu-id="fa4c1-129">Cosmos に保存されている Microsoft 365 データへのアクセスは、承認された担当者に制限されます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-129">Access to Microsoft 365 data stored in Cosmos is restricted to authorized personnel.</span></span> <span data-ttu-id="fa4c1-130">Microsoft 365 では、監査機能の管理を、監査機能を担当するサービス チーム メンバーの限定されたサブセットに制限しています。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-130">Microsoft 365 restricts the management of audit functionality to the limited subset of service team members that are responsible for audit functionality.</span></span> <span data-ttu-id="fa4c1-131">これらのチーム メンバーは、Cosmos のデータを変更または削除する機能を持ってはいではなく、Cosmos のログ記録メカニズムに対する変更はすべて記録および監査されます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-131">These team members do not have the ability to modify or delete data from Cosmos, and all changes to logging mechanisms for Cosmos are recorded and audited.</span></span> <span data-ttu-id="fa4c1-132">監査ログは、インシデント調査をサポートし、規制要件を満たすのに十分な長い期間保持されます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-132">Audit logs are retained long enough to support incident investigations and meet regulatory requirements.</span></span> <span data-ttu-id="fa4c1-133">Cosmos での監査ログ データ保持の正確な期間は、サービス チームによって決定されます。ほとんどの監査ログ データは 90 日間以上保持されます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-133">The exact period of audit log data retention in Cosmos is determined by the service teams; most audit log data is retained for 90 days or longer.</span></span>

## <a name="how-does-microsoft-365-protect-end-user-identifiable-information-that-may-be-captured-in-audit-logs"></a><span data-ttu-id="fa4c1-134">Microsoft 365 では、監査ログに記録される可能性があるエンド ユーザーを特定できる情報を保護する方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-134">How does Microsoft 365 protect end-user identifiable information that may be captured in audit logs?</span></span>

<span data-ttu-id="fa4c1-135">Cosmos にデータをアップロードする前に、ODL アプリケーションはスクラブ サービスを使用して、テナント情報やエンド ユーザーを特定できる情報などの顧客データを含むフィールドを難読化し、それらのフィールドをハッシュ値に置き換える。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-135">Prior to uploading data into Cosmos, the ODL application uses a scrubbing service to obfuscate any fields that contain customer data, such as tenant information and end-user identifiable information, and replace those fields with a hash value.</span></span> <span data-ttu-id="fa4c1-136">匿名化されたログとハッシュされたログが書き換え、Cosmos にアップロードされます。</span><span class="sxs-lookup"><span data-stu-id="fa4c1-136">The anonymized and hashed logs are rewritten and then uploaded into Cosmos.</span></span>
