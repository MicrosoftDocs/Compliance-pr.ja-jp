## <a name="how-does-microsoft-365-centralize-and-report-on-audit-logs"></a><span data-ttu-id="37800-120">Microsoft 365 では、監査ログを一元化してレポートする方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="37800-120">How does Microsoft 365 centralize and report on audit logs?</span></span>

<span data-ttu-id="37800-121">さまざまな種類のログ データが Microsoft 365 サーバーから Cosmos と呼ばれる内部のビッグ データ コンピューティング サービスにアップロードされます。</span><span class="sxs-lookup"><span data-stu-id="37800-121">Many different types of log data are uploaded from Microsoft 365 servers to an internal, big data computing service called Cosmos.</span></span> <span data-ttu-id="37800-122">各サービス チームは、それぞれのサーバーから Cosmos データベースに監査ログをアップロードして、集約と分析を行います。</span><span class="sxs-lookup"><span data-stu-id="37800-122">Each service team uploads audit logs from their respective servers into the Cosmos database for aggregation and analysis.</span></span> <span data-ttu-id="37800-123">このデータ転送は、Office Data Loader (ODL) と呼ばれる独自のオートメーション ツールを使用して、承認済みポートおよびプロトコル上の FIPS 140-2 検証済み TLS 接続を使用して行われます。</span><span class="sxs-lookup"><span data-stu-id="37800-123">This data transfer occurs over a FIPS 140-2-validated TLS connection on approved ports and protocols using a proprietary automation tool called the Office Data Loader (ODL).</span></span>

## <a name="how-does-microsoft-365-protect-audit-logs"></a><span data-ttu-id="37800-127">Microsoft 365 は監査ログを保護する方法を説明します。</span><span class="sxs-lookup"><span data-stu-id="37800-127">How does Microsoft 365 protect audit logs?</span></span>

<span data-ttu-id="37800-128">Microsoft 365 で監査レコードの収集と処理に使用されるツールでは、元の監査レコードのコンテンツや時間の順序に対する永続的または不可逆的な変更は許可されません。</span><span class="sxs-lookup"><span data-stu-id="37800-128">The tools used in Microsoft 365 to collect and process audit records do not allow permanent or irreversible changes to the original audit record content or time ordering.</span></span> <span data-ttu-id="37800-129">Cosmos に格納されている Microsoft 365 データへのアクセスは、承認された担当者に制限されます。</span><span class="sxs-lookup"><span data-stu-id="37800-129">Access to Microsoft 365 data stored in Cosmos is restricted to authorized personnel.</span></span> <span data-ttu-id="37800-130">Microsoft 365 は、監査機能の管理を、監査機能を担当するサービス チーム メンバーの限定されたサブセットに制限します。</span><span class="sxs-lookup"><span data-stu-id="37800-130">Microsoft 365 restricts the management of audit functionality to the limited subset of service team members that are responsible for audit functionality.</span></span> <span data-ttu-id="37800-131">これらのチーム メンバーは、Cosmos からデータを変更または削除する機能を持たなく、Cosmos のログメカニズムに対する変更はすべて記録および監査されます。</span><span class="sxs-lookup"><span data-stu-id="37800-131">These team members do not have the ability to modify or delete data from Cosmos, and all changes to logging mechanisms for Cosmos are recorded and audited.</span></span> <span data-ttu-id="37800-132">監査ログは、インシデント調査をサポートし、規制要件を満たすのに十分な時間保持されます。</span><span class="sxs-lookup"><span data-stu-id="37800-132">Audit logs are retained long enough to support incident investigations and meet regulatory requirements.</span></span> <span data-ttu-id="37800-133">Cosmos の監査ログ データ保持の正確な期間は、サービス チームによって決まります。ほとんどの監査ログ データは 90 日以上保持されます。</span><span class="sxs-lookup"><span data-stu-id="37800-133">The exact period of audit log data retention in Cosmos is determined by the service teams; most audit log data is retained for 90 days or longer.</span></span>

## <a name="how-does-microsoft-365-protect-end-user-identifiable-information-that-may-be-captured-in-audit-logs"></a><span data-ttu-id="37800-134">Microsoft 365 は、監査ログに記録される可能性があるエンド ユーザー識別可能な情報を保護する方法を説明します。</span><span class="sxs-lookup"><span data-stu-id="37800-134">How does Microsoft 365 protect end-user identifiable information that may be captured in audit logs?</span></span>

<span data-ttu-id="37800-135">データを Cosmos にアップロードする前に、ODL アプリケーションはスクラブ サービスを使用して、テナント情報やエンドユーザー識別可能な情報など、顧客データを含むフィールドを難読化し、それらのフィールドをハッシュ値に置き換える。</span><span class="sxs-lookup"><span data-stu-id="37800-135">Prior to uploading data into Cosmos, the ODL application uses a scrubbing service to obfuscate any fields that contain customer data, such as tenant information and end-user identifiable information, and replace those fields with a hash value.</span></span> <span data-ttu-id="37800-136">匿名化ログとハッシュ ログが書き換え、次に Cosmos にアップロードされます。</span><span class="sxs-lookup"><span data-stu-id="37800-136">The anonymized and hashed logs are rewritten and then uploaded into Cosmos.</span></span>
