## <a name="response-management-process"></a><span data-ttu-id="4934e-140">応答管理プロセス</span><span class="sxs-lookup"><span data-stu-id="4934e-140">Response management process</span></span>

<span data-ttu-id="4934e-141">Microsoft 365 セキュリティ チームとサービス チームは、NIST 800-61 の応答管理フェーズに基づいて、セキュリティ インシデントに対して同じアプローチを取り組み、取り組む。</span><span class="sxs-lookup"><span data-stu-id="4934e-141">The Microsoft 365 Security team and the service teams work together on and take the same approach to security incidents, which is based on the NIST 800-61 response management phases:</span></span>

- <span data-ttu-id="4934e-142">**準備**: ツール、プロセス、コンピテンシー、準備など、対応するために必要な組織の準備を指します。</span><span class="sxs-lookup"><span data-stu-id="4934e-142">**Preparation**: Refers to the organizational preparation that is needed to be able to respond, including tools, processes, competencies, and readiness.</span></span>
- <span data-ttu-id="4934e-143">**検出&** 分析 : 実稼働環境でセキュリティ インシデントを検出し、すべてのイベントを分析してセキュリティ インシデントの信頼性を確認するアクティビティを参照します。</span><span class="sxs-lookup"><span data-stu-id="4934e-143">**Detection & analysis**: Refers to the activity to detect a security incident in a production environment and to analyze all events to confirm the authenticity of the security incident.</span></span>
