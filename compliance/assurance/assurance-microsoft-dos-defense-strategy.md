## <a name="core-principles-of-defense-against-denial-of-service-attacks"></a><span data-ttu-id="37b8b-104">サービス拒否攻撃に対する防御の主要な原則</span><span class="sxs-lookup"><span data-stu-id="37b8b-104">Core principles of defense against denial-of-service attacks</span></span>

<span data-ttu-id="37b8b-105">ネットワーク ベースのサービス拒否 (DoS) 攻撃に対する防御には、吸収、検出、軽減の 3 つの主要な原則があります。</span><span class="sxs-lookup"><span data-stu-id="37b8b-105">There are three core principles when defending against network-based denial-of-service (DoS) attacks: absorption, detection, and mitigation.</span></span> <span data-ttu-id="37b8b-106">吸収は検出の前に行われるので、軽減策を開始する前に検出を行う必要があります。</span><span class="sxs-lookup"><span data-stu-id="37b8b-106">Absorption happens before detection, and detection must occur before mitigation can begin.</span></span> <span data-ttu-id="37b8b-107">小さな DoS 攻撃を吸収できない場合、サービスは攻撃を検出するのに十分な時間を生き残る必要があります。</span><span class="sxs-lookup"><span data-stu-id="37b8b-107">If even a small DoS attack cannot be absorbed, then services will not survive long enough for the attack to be detected.</span></span> <span data-ttu-id="37b8b-108">システムが圧倒される前に攻撃を検出すると、防御側は応答計画を実装できます。</span><span class="sxs-lookup"><span data-stu-id="37b8b-108">By detecting an attack before the system is overwhelmed, defenders can implement a response plan.</span></span>

- <span data-ttu-id="37b8b-115">攻撃を検出する時間を減らします。</span><span class="sxs-lookup"><span data-stu-id="37b8b-115">Decrease the time to detect an attack.</span></span>

<span data-ttu-id="37b8b-116">Microsoft クラウド サービスを使用するセキュリティ上の利点の 1 つは、大規模な Microsoft サービスが、コスト効率の高い方法でクラウド顧客に強力なネットワーク保護を提供する方法です。</span><span class="sxs-lookup"><span data-stu-id="37b8b-116">One security benefit of using Microsoft cloud services is how large-scale Microsoft services provide strong network protection to cloud customers in a cost-effective manner.</span></span> <span data-ttu-id="37b8b-117">大規模な場合でも、吸収、検出、軽減のバランスが必要です。</span><span class="sxs-lookup"><span data-stu-id="37b8b-117">Even on a large scale, there must be a balance between absorption, detection, and mitigation.</span></span> <span data-ttu-id="37b8b-118">そのバランスを見つけるために、Microsoft は、Microsoft サービスが吸収する必要がある量を推定するために、成長率を攻撃します。</span><span class="sxs-lookup"><span data-stu-id="37b8b-118">To find that balance, Microsoft studies attack growth rates to estimate how much Microsoft services need to absorb.</span></span>
