## <a name="what-role-does-encryption-play-in-protecting-customer-content"></a><span data-ttu-id="760bc-104">お客様のコンテンツを保護する際に、暗号化はどのような役割を果たしますか?</span><span class="sxs-lookup"><span data-stu-id="760bc-104">What role does encryption play in protecting customer content?</span></span>

<span data-ttu-id="760bc-105">ほとんどの Microsoft business cloud services はマルチテナントです。つまり、お客様のコンテンツは他のお客様と同じ物理ハードウェアに格納されることがあります。</span><span class="sxs-lookup"><span data-stu-id="760bc-105">Most Microsoft business cloud services are multitenant, meaning that customer content may be stored on the same physical hardware as that of other customers.</span></span> <span data-ttu-id="760bc-106">お客様のコンテンツの機密性を保護するために、Microsoft 365 は、すべてのデータを保存中に暗号化し、最も強力で最も安全な暗号化プロトコルを使用して転送中に暗号化します。</span><span class="sxs-lookup"><span data-stu-id="760bc-106">To protect the confidentiality of customer content, Microsoft 365 encrypts all data at rest and in transit with some of the strongest and most secure encryption protocols available.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-at-rest"></a><span data-ttu-id="760bc-110">Microsoft 365 は、保存データを暗号化する方法を教えてください。</span><span class="sxs-lookup"><span data-stu-id="760bc-110">How does Microsoft 365 encrypt data-at-rest?</span></span>

<span data-ttu-id="760bc-111">Microsoft 365 のすべてのお客様のコンテンツは、1つまたは複数の形式の暗号化によって保護されています。</span><span class="sxs-lookup"><span data-stu-id="760bc-111">All customer content in Microsoft 365 is protected by one or more forms of encryption.</span></span> <span data-ttu-id="760bc-112">Microsoft サーバーは、BitLocker を使用して、ボリュームレベルで顧客コンテンツを含むディスクドライブを暗号化します。</span><span class="sxs-lookup"><span data-stu-id="760bc-112">Microsoft servers use BitLocker to encrypt the disk drives containing customer content at the volume-level.</span></span> <span data-ttu-id="760bc-113">BitLocker によって提供される暗号化は、ユーザーのコンテンツを含むディスクへの不正な物理的なアクセスにつながる可能性がある他のプロセスやコントロールで lapses が発生した場合にお客様のコンテンツを保護します (たとえば、アクセス制御やハードウェアのリサイクル)</span><span class="sxs-lookup"><span data-stu-id="760bc-113">The encryption provided by BitLocker protects customer content in case of lapses in other processes or controls (for example, access control or recycling of hardware) that could lead to unauthorized physical access to disks containing customer content.</span></span>

<span data-ttu-id="760bc-114">Exchange Online、Microsoft Teams、SharePoint Online、および OneDrive for Business では、アプリケーション層でサービス暗号化を使用して、顧客コンテンツを暗号化します。</span><span class="sxs-lookup"><span data-stu-id="760bc-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="760bc-115">サービス暗号化は、強力な暗号化保護の上に権限の保護機能と管理機能を提供します。</span><span class="sxs-lookup"><span data-stu-id="760bc-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="760bc-116">また、Windows オペレーティングシステムと、これらのオペレーティングシステムによって保存または処理されるお客様のデータを分離することもできます。</span><span class="sxs-lookup"><span data-stu-id="760bc-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="760bc-117">Microsoft 365 は、送信データを暗号化する方法を教えてください。</span><span class="sxs-lookup"><span data-stu-id="760bc-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="760bc-118">Microsoft 365 製品とサービスでは、TLS などの強力なトランスポートプロトコルを使用して、許可されていない第三者がネットワーク上を移動している間に顧客データを傍受できないようにします。</span><span class="sxs-lookup"><span data-stu-id="760bc-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="760bc-119">転送中のデータの例としては、配信処理中のメールメッセージ、オンライン会議で行われた会話、データセンター間でレプリケートされるファイルなどがあります。</span><span class="sxs-lookup"><span data-stu-id="760bc-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="760bc-120">Microsoft 365 では、ユーザーのデバイスが Microsoft サーバーと通信している場合、または Microsoft サーバーが別のサーバーと通信している場合は、データは「転送中」と見なされます。</span><span class="sxs-lookup"><span data-stu-id="760bc-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="760bc-121">Exchange Online、SharePoint Online、Microsoft Teams、および Office Online はすべて TLS を使用して、転送中のデータを機密に保つことができます。</span><span class="sxs-lookup"><span data-stu-id="760bc-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="760bc-122">Microsoft 365 は、暗号化に使用されるキーをどのように管理していますか?</span><span class="sxs-lookup"><span data-stu-id="760bc-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="760bc-123">強力な暗号化は、データの暗号化に使用されるキーと同じようにセキュリティで保護されています。</span><span class="sxs-lookup"><span data-stu-id="760bc-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="760bc-124">Microsoft では、独自のセキュリティ証明書を使用して、送信データの TLS 接続を暗号化しています。</span><span class="sxs-lookup"><span data-stu-id="760bc-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="760bc-125">データを保存する場合、BitLocker で保護されたボリュームは、ボリュームマスターキーで暗号化された完全なボリューム暗号化キーを使用して暗号化されます。これは、サーバーのトラステッドプラットフォームモジュール (TPM) にバインドされます。</span><span class="sxs-lookup"><span data-stu-id="760bc-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="760bc-126">BitLocker は FIPS に準拠したアルゴリズムを使用して、暗号化キーが暗号化されずに保存または送信されることがないようにします。</span><span class="sxs-lookup"><span data-stu-id="760bc-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="760bc-127">サービス暗号化では、Exchange Online、Microsoft Teams、SharePoint Online、OneDrive for business に保存された、お客様のデータに対する暗号化の層が追加されています。</span><span class="sxs-lookup"><span data-stu-id="760bc-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="760bc-128">サービスの暗号化では、暗号化キーの管理に関して、Microsoft が管理するキーまたは顧客キーという2つのオプションが提供されます。</span><span class="sxs-lookup"><span data-stu-id="760bc-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="760bc-129">Microsoft 管理キーを使用する場合、Microsoft 365 サービスは、サービスの暗号化に使用されるルートキーを自動的に生成し、安全に保存します。</span><span class="sxs-lookup"><span data-stu-id="760bc-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="760bc-130">独自のルート暗号化キーを制御する要件を持つお客様は、お客様のキーを使用してサービス暗号化を利用できます。</span><span class="sxs-lookup"><span data-stu-id="760bc-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="760bc-131">顧客キーを使用すると、オンプレミスのハードウェアサービスモジュール (HSM) または Azure Key Vault (AKV) を使用して、独自の暗号化キーを生成できます。</span><span class="sxs-lookup"><span data-stu-id="760bc-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="760bc-132">顧客のルートキーは、ユーザーのメールボックスのデータまたはファイルを暗号化する keychains の1つのルートとして使用できる AKV に格納されます。</span><span class="sxs-lookup"><span data-stu-id="760bc-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="760bc-133">顧客のルートキーには、データ暗号化用の Microsoft 365 サービスコードによる間接的なアクセスのみが可能で、Microsoft の従業員が直接アクセスすることはできません。</span><span class="sxs-lookup"><span data-stu-id="760bc-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
