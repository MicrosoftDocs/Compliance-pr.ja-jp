## <a name="what-role-does-encryption-play-in-protecting-customer-content"></a><span data-ttu-id="19d82-104">顧客のコンテンツを保護する上で暗号化はどのような役割を果たしますか?</span><span class="sxs-lookup"><span data-stu-id="19d82-104">What role does encryption play in protecting customer content?</span></span>

<span data-ttu-id="19d82-105">ほとんどの Microsoft ビジネス クラウド サービスはマルチテナントです。つまり、顧客のコンテンツは、他のお客様と同じ物理ハードウェアに保存される可能性があります。</span><span class="sxs-lookup"><span data-stu-id="19d82-105">Most Microsoft business cloud services are multitenant, meaning that customer content may be stored on the same physical hardware as that of other customers.</span></span> <span data-ttu-id="19d82-106">お客様のコンテンツの機密性を保護するために、Microsoft 365 は、利用可能な最も強力で最も安全な暗号化プロトコルを使用して、保存中および転送中のすべてのデータを暗号化します。</span><span class="sxs-lookup"><span data-stu-id="19d82-106">To protect the confidentiality of customer content, Microsoft 365 encrypts all data at rest and in transit with some of the strongest and most secure encryption protocols available.</span></span>

<span data-ttu-id="19d82-107">暗号化は、強力なアクセス制御の代わりに使用されるのではありません。</span><span class="sxs-lookup"><span data-stu-id="19d82-107">Encryption is not a substitute for strong access controls.</span></span> <span data-ttu-id="19d82-108">Microsoft 365 のアクセス制御ポリシーである Zero Standing Access (ZSA) は、Microsoft の従業員による不正なアクセスから顧客コンテンツを保護します。</span><span class="sxs-lookup"><span data-stu-id="19d82-108">Microsoft 365's access control policy of Zero Standing Access (ZSA) protects customer content from unauthorized access by Microsoft employees.</span></span> <span data-ttu-id="19d82-109">暗号化は、保存されている場所で顧客のコンテンツの機密性を保護し、Microsoft 365 システム間または Microsoft 365 と顧客との間で転送中にコンテンツが読み取らなされるのを防ぐことで、アクセス制御を補完します。</span><span class="sxs-lookup"><span data-stu-id="19d82-109">Encryption complements access control by protecting the confidentiality of customer content wherever it is stored and by preventing content from being read while in transit between Microsoft 365 systems or between Microsoft 365 and the customer.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-at-rest"></a><span data-ttu-id="19d82-110">Microsoft 365 は、保存データを暗号化する方法を説明します。</span><span class="sxs-lookup"><span data-stu-id="19d82-110">How does Microsoft 365 encrypt data-at-rest?</span></span>

<span data-ttu-id="19d82-111">Microsoft 365 のすべての顧客コンテンツは、1 つ以上の形式の暗号化によって保護されます。</span><span class="sxs-lookup"><span data-stu-id="19d82-111">All customer content in Microsoft 365 is protected by one or more forms of encryption.</span></span> <span data-ttu-id="19d82-112">Microsoft サーバーは、BitLocker を使用して、顧客のコンテンツを含むディスク ドライブをボリューム レベルで暗号化します。</span><span class="sxs-lookup"><span data-stu-id="19d82-112">Microsoft servers use BitLocker to encrypt the disk drives containing customer content at the volume-level.</span></span> <span data-ttu-id="19d82-113">BitLocker によって提供される暗号化は、顧客のコンテンツを含むディスクへの不正な物理的アクセスにつながる可能性がある他のプロセスまたは制御 (アクセス制御やハードウェアのリサイクルなど) が失効した場合に、お客様のコンテンツを保護します。</span><span class="sxs-lookup"><span data-stu-id="19d82-113">The encryption provided by BitLocker protects customer content in case of lapses in other processes or controls (for example, access control or recycling of hardware) that could lead to unauthorized physical access to disks containing customer content.</span></span>

<span data-ttu-id="19d82-114">Exchange Online、Microsoft Teams、SharePoint Online、OneDrive for Business は、アプリケーションレイヤーで Service Encryption を使用して顧客コンテンツを暗号化します。</span><span class="sxs-lookup"><span data-stu-id="19d82-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="19d82-115">Service Encryption は、強力な暗号化保護の上に権利保護と管理機能を提供します。</span><span class="sxs-lookup"><span data-stu-id="19d82-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="19d82-116">また、Windows オペレーティング システムと、それらのオペレーティング システムによって保存または処理される顧客データを分離できます。</span><span class="sxs-lookup"><span data-stu-id="19d82-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="19d82-117">Microsoft 365 は、転送中のデータを暗号化する方法を説明します。</span><span class="sxs-lookup"><span data-stu-id="19d82-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="19d82-118">Microsoft 365 製品とサービスは、TLS などの強力なトランスポート プロトコルを使用して、未承認の関係者がネットワークを越える移動中に顧客データを盗聴するのを防ぐ。</span><span class="sxs-lookup"><span data-stu-id="19d82-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="19d82-119">送信中のデータの例としては、配信中のメール メッセージ、オンライン会議で行う会話、データセンター間でレプリケートされるファイルがあります。</span><span class="sxs-lookup"><span data-stu-id="19d82-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="19d82-120">Microsoft 365 では、ユーザーのデバイスが Microsoft サーバーと通信している場合や、Microsoft サーバーが別のサーバーと通信している場合、データは "送信中" と見なされます。</span><span class="sxs-lookup"><span data-stu-id="19d82-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="19d82-121">Exchange Online、SharePoint Online、Microsoft Teams、および Office Online は、すべて TLS を使用して、転送中もデータを機密に維持します。</span><span class="sxs-lookup"><span data-stu-id="19d82-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="19d82-122">Microsoft 365 では、暗号化に使用されるキーを管理する方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="19d82-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="19d82-123">強力な暗号化は、データの暗号化に使用されるキーと同じだけ安全です。</span><span class="sxs-lookup"><span data-stu-id="19d82-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="19d82-124">Microsoft は、独自のセキュリティ証明書を使用して、転送中のデータの TLS 接続を暗号化します。</span><span class="sxs-lookup"><span data-stu-id="19d82-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="19d82-125">保存データの場合、BitLocker で保護されたボリュームはボリューム全体の暗号化キーで暗号化されます。この暗号化キーはボリューム マスター キーで暗号化され、ボリューム マスター キーはサーバーのトラステッド プラットフォーム モジュール (TPM) にバインドされます。</span><span class="sxs-lookup"><span data-stu-id="19d82-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="19d82-126">BitLocker は FIPS 準拠のアルゴリズムを使用して、暗号化キーがクリアでネットワークに保存または送信されるのを確実にします。</span><span class="sxs-lookup"><span data-stu-id="19d82-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="19d82-127">Service Encryption は、Exchange Online、Microsoft Teams、SharePoint Online、OneDrive for Business で保存中の顧客データの暗号化の追加レイヤーを提供します。</span><span class="sxs-lookup"><span data-stu-id="19d82-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="19d82-128">サービスの暗号化を使用すると、Microsoft で管理されるキーと顧客キーの 2 つの暗号化キー管理のオプションが提供されます。</span><span class="sxs-lookup"><span data-stu-id="19d82-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="19d82-129">Microsoft が管理するキーを使用する場合、Microsoft 365 サービスはサービスの暗号化に使用されるルート キーを自動的に生成し、安全に保存します。</span><span class="sxs-lookup"><span data-stu-id="19d82-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="19d82-130">独自のルート暗号化キーを制御する要件を持つお客様は、顧客キーを使用してサービスの暗号化を利用できます。</span><span class="sxs-lookup"><span data-stu-id="19d82-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="19d82-131">顧客キーを使用すると、顧客はオンプレミスのハードウェア サービス モジュール (VAULT) または Azure Key Vault (A VAULT) を使用して、独自の暗号化キーを生成できます。</span><span class="sxs-lookup"><span data-stu-id="19d82-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="19d82-132">顧客のルート キーは ARK に格納され、顧客のメールボックス のデータまたはファイルを暗号化するキーチェーンのルートとして使用できます。</span><span class="sxs-lookup"><span data-stu-id="19d82-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="19d82-133">顧客のルート キーは、データ暗号化のために Microsoft 365 サービス コードによって間接的にのみアクセスできます。また、Microsoft の従業員が直接アクセスすることはできません。</span><span class="sxs-lookup"><span data-stu-id="19d82-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
