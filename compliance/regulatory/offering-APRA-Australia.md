# <a name="australian-prudential-regulation-authority-apra"></a>オーストラリア Prudential Regulation Authority (APRA)

## <a name="apra-overview"></a>APRA の概要

[オーストラリアの Prudential Regulation Authority](https://www.apra.gov.au/) (APRA) は、オーストラリアの銀行、信用組合、保険会社、その他の金融サービス機関を監督します。 クラウド コンピューティングの推進力を認識し、APRA は規制対象企業に対して、効果的なガバナンス、徹底的なリスク評価、および定期的な保証プロセスを備えた、慎重なクラウド導入戦略を実施する必要があります。 規制対象機関は、重要なビジネス 活動をアウトソーシングする際に、APRA [Prudential Standard CPS 231 Outsourcing](https://www.apra.gov.au/sites/default/files/Prudential-Standard-CPS-231-Outsourcing-%28July-2017%29.pdf) に準拠する必要があります。中断された場合、金融機関の事業運営やリスクを効果的に管理する能力に大きな影響を与える可能性のある活動。 APRA に提出されたクラウド コンピューティング サービスに関するアウトソーシング契約のレビューに基づいて、APRA は情報ホワイト ペーパー[](https://www.apra.gov.au/sites/default/files/information_paper_-_outsourcing_involving_cloud_computing_services.pdf)に詳細なガイダンスを公開しました。規制対象エンティティがクラウド プロバイダーとサービスを効果的に評価し、クラウドへのアウトソーシングに関する規制上の問題を通じてそれらをガイドするために、クラウド コンピューティング サービスを含むアウトソーシング。  クラウド サービスへのアウトソーシングを行う場合、規制対象機関は、APRA [Prudential Standard CPS 234 Information Security](https://www.legislation.gov.au/Details/F2018L01745)への継続的な準拠を確認し、検討する必要があります。

## <a name="microsoft-response-to-the-apra-information-paper-on-cloud"></a>クラウド上の APRA 情報用紙に対する Microsoft の対応

この Microsoft ホワイト ペーパーでは、クラウド コンピューティング サービスを含む APRA 情報紙アウトソーシングで発生した各問題に対する詳細な対応を提供し、金融サービスに関する詳細なガイダンス [を提供します](https://www.apra.gov.au/sites/default/files/information_paper_-_outsourcing_involving_cloud_computing_services.pdf)。 APRA のガイドラインでは、クラウドの使用状況が通常、低、高く、極端な固有のリスクとなる 3 つのリスク カテゴリを特定し、規制対象エンティティがリスク評価の一部として考慮する必要がある重要な問題を強調します。

Microsoft の対応では、最もリスクの高い 2 つのカテゴリに焦点を当てしています。 クラウド サービスはリスク カテゴリによって禁止されませんが、APRA は、リスク カテゴリを上に移動するに伴い、非常に高いレベルの注意を引き受ける必要があります。また、リスク カテゴリを上に進めるには、APRA の調査レベルが上がると予想されます。 APRA では、一般的にクラウド アウトソーシングに固有の高または極端なリスクを示すさまざまな要因がリストされています。 Microsoft は、これらの各要因に詳細に対応し、データとワークロードを Azure に移動するリスクを評価および管理するのに役立つ情報とツールを提供します。

また、Microsoft では、戦略、ガバナンス、ソリューション選択プロセス、APRA アクセスと行動能力、移行アプローチ、リスク評価とセキュリティ、継続的な監視、業務中断、監査と保証など、各 APRA リスク管理の考慮事項にも対応しています。 Azure を展開する際に各問題に対応するためのアドバイスやツールを提供します。

APRA 規制に準拠してデータとワークロードを Azure に移行する実用的なサポートを得る: クラウド上の APRA 情報ペーパーに対する Microsoft の対応 [をダウンロードします](https://aka.ms/navigatecloudaustralia)。

## <a name="microsoft-response-to-the-apra-cps-234-on-information-security"></a>情報セキュリティに関する APRA CPS 234 に対する Microsoft の対応

APRA [Prudential Standard CPS 234 Information Security では](https://www.legislation.gov.au/Details/F2018L01745) 、規制機関は次の作業を行う必要があります。

- 情報セキュリティ関連の役割と責任を明確に定義する。
- 情報資産に対する脅威の規模と程度に合った情報セキュリティ機能を維持する。
- 情報資産を保護するためのコントロールを実装し、コントロールの有効性を定期的にテストおよび保証します。そして
- 重要な情報セキュリティ インシデントを APRA に速やかに通知します。

CPS 234 は、Microsoft のコア セキュリティ フレームワーク (保護、検出、対応) を密接にミラー化します。

[Microsoft クラウド サービス: APRA Prudential Standard CPS 234 情報](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE2OsZg) セキュリティへの準拠は、関連する CPS 234 の規制義務を定め、MICROSOFT クラウド サービスのコントロール、機能、機能、契約コミットメント、およびサポート情報に対応して、APRA 規制対象エンティティが CPS 234 に基づく規制上の義務に準拠するのに役立ちます。

## <a name="navigating-your-way-to-the-cloud-a-compliance-checklist-for-financial-institutions-in-australia"></a>クラウドへの移行: オーストラリアの金融機関向けコンプライアンス チェックリスト

この Microsoft チェックリストでは、金融機関がクラウドに移行するときに対応する必要がある APRA の規制要件について説明します。 これは [、Prudential Standard CPS 231 アウト](https://www.apra.gov.au/sites/default/files/Prudential-Standard-CPS-231-Outsourcing-%28July-2017%29.pdf)ソーシングだけでなく、ビジネス継続性やリスク管理など、その他の関連する APRA 標準に対して Azure をマップします。 このチェックリストを完了すると、金融機関は関連する APRA 要件を満たしていることを信頼して Azure を導入できます。

クラウドにおけるリスク 保証に対する包括的なアプローチに依存することにより、オーストラリアの金融サービス組織は、APRA ガイダンスと一貫性があるだけでなく、オンプレミスまたは他のホストされたソリューションよりも高度なセキュリティ リスク管理プロファイルを顧客に提供できる方法で Microsoft クラウド サービスに移行できると確信しています。

APRA の規制に準拠してデータとワークロードを Azure に移行するための実践的なサポートを得る: Microsoft クラウド サービスをダウンロード [する:](https://www.microsoft.com/cms/api/am/binary/RE3ez0C)オーストラリアの金融機関向けコンプライアンス チェックリスト。

## <a name="frequently-asked-questions"></a>よく寄せられる質問

**金融機関は重要なビジネス 活動をアウトソーシングする前に APRA の承認を必要としていますか?**

いいえ。 ただし、ほとんどの規制対象の金融機関は、オーストラリア内の重要なビジネス 活動をアウトソーシングする契約を締結した後、APRA に通知するか、またはオーストラリア以外の活動をアウトソーシングする前に APRA に相談する必要があります。

さらに、クラウド サービスがクラウド上の [APRA](https://www.apra.gov.au/sites/default/files/information_paper_-_outsourcing_involving_cloud_computing_services.pdf)情報ペーパーに記載されている「高いリスクまたは極端な固有のリスク」を実施すると判断された場合、金融機関は、サービスがオーストラリア内で提供されているのか外部で提供されているのかにかかわらず、APRA に相談するように奨励されます (ただし、必須ではありません)。

**オーストラリア以外でのデータの転送は許可されますか?**

はい。 一般的なプライバシーに関する法律 (金融機関に対してではなく、すべてのセクターに適用されます) は、特定の条件下でオーストラリア外への転送を許可します。 Microsoft は、Microsoft クラウド サービスを使用する際にオーストラリア以外のデータの転送が許可される、オーストラリアのプライバシー原則に沿った契約条件に同意します。 ただし、オーストラリアの金融サービスのお客様の多くは、オーストラリアのデータセンターから利用できるクラウド サービスを利用しています。このクラウド サービスでは、オーストラリアの地域に保管されているデータのカテゴリを格納する特定の契約上のコミットメントを行っています。 これらの取り組みについては、コンプライアンス チェックリストで [詳しい説明を示します](https://www.microsoft.com/cms/api/am/binary/RE3ez0C)。

## <a name="use-microsoft-compliance-manager-to-assess-your-risk"></a>Microsoft コンプライアンス マネージャーを使用してリスクを評価する

[Microsoft コンプライアンス マネージャー](/microsoft-365/compliance/compliance-manager)は、[ Microsoft 365 コンプライアンス センター](/microsoft-365/compliance/microsoft-365-compliance-center)の機能で、組織のコンプライアンスに対する姿勢を把握し、リスクを軽減するための処置を実行できるようにします。 コンプライアンスマネージャーには、この規制の評価を構築するためのプレミアム テンプレートが用意されています。 コンプライアンスマネージャーの **評価テンプレート** ページでテンプレートを見つけます。 [コンプライアンスマネージャーで評価をする方法](/microsoft-365/compliance/compliance-manager-assessments)について説明します。
